J's Counting

A Discord counting bot, but with a chaotic twist





You've probably seen what a Discord counting bot is meant to look like. The bot is added a server, designates one channel as the counting channel, and keeps track of the number for that server, independently. Each server it's in will reach a different number, and each number is kept track of separately.

That's not what we do here.

With this bot, the number is stored in a single file, which determines the number for every server the bot is in. For example, if a person in one server counts from 0 to 1, the bot saves the number as 1. The next person to count has to count 2, or else the bot flags them as wrong. The other servers, however, will not know the actual number that is being tracked without using the /current command.

The more servers the bot can participate in, the more chaos arises from everyone having to check the number at once before counting. A game that requires high audience retention, but could bear high entertainment if executed correctly.
